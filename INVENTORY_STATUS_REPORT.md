## ⚠️ PROBLEMAS IDENTIFICADOS

### 1. Paginación No Funciona Correctamente

**Problema**: Los botones de paginación están implementados pero pueden no funcionar si el backend no devuelve la estructura esperada.

**Componentes afectados**:
- `inventory-list.component.ts` (Productos)
- `dish-management.component.ts` (Platos)
- `drink-management.component.ts` (Bebidas)
- `addition-management.component.ts` (Adiciones)

**Código actual**:
```typescript
loadProducts(): void {
  this.productService.getProducts(this.currentPage).subscribe({
    next: (response) => {
      if (!response.error) {
        const message = response.message as any;
        if (message && message.content) {
          this.products = message.content;
          this.totalPages = message.totalPages || 1;
        } else if (Array.isArray(message)) {
          // Fallback si la respuesta es un array directo
          this.products = message;
          this.totalPages = message.length > 0 ? 1 : 0;
        }
      }
    }
  });
}
```

**Problema detectado**:
1. El backend devuelve `ApiResponse<ProductListResponse[]>` según la especificación
2. El código espera `message.content` y `message.totalPages`
3. Pero según la especificación, el backend devuelve directamente el array en `message`
4. **No hay información de paginación en la respuesta del backend**

**Solución necesaria**:

**Opción A - Modificar Backend** (Recomendado):
El backend debería devolver:
```json
{
  "message": {
    "content": [...],
    "totalPages": 5,
    "totalElements": 50,
    "size": 10,
    "number": 0
  },
  "error": false
}
```

**Opción B - Modificar Frontend**:
Si el backend solo devuelve el array, la paginación debe manejarse diferente:
```typescript
// Asumir que cada página tiene 10 elementos
// Si devuelve menos de 10, es la última página
loadProducts(): void {
  this.productService.getProducts(this.currentPage).subscribe({
    next: (response) => {
      if (!response.error && Array.isArray(response.message)) {
        this.products = response.message;
        // Si devuelve menos de 10, es la última página
        this.hasNextPage = response.message.length === 10;
      }
    }
  });
}
```

### 2. Índice de Paginación (0-based vs 1-based)

**Problema**: El backend usa paginación 0-indexed (página 0 es la primera), pero el frontend muestra `currentPage = 1`.

**Código actual**:
```typescript
currentPage = 1; // Frontend muestra "Página 1"
this.productService.getProducts(this.currentPage); // Envía 1 al backend
```

**Solución**:
```typescript
// Opción 1: Ajustar al enviar
this.productService.getProducts(this.currentPage - 1);

// Opción 2: Usar currentPage = 0 internamente y mostrar currentPage + 1 en UI
```

### 3. Falta Manejo de Imágenes por Defecto

**Problema**: Algunos componentes no tienen imagen placeholder cuando `photo` es null.

**Solución**: Ya implementado en `inventory-list` pero falta en otros componentes:
```typescript
onImageError(event: Event): void {
  const img = event.target as HTMLImageElement;
  img.src = 'assets/images/no-image.png';
}
```
